    Unity3D插件使用文档

    简介

    本文档将帮助你如何使用游戏引擎 Unity 3D 读取魔迅MotionStudio动捕软件的运动捕捉数据。 MotionStudio软件支持实时传输实时和离线动捕数据，实时数据是指连接设备时的数据，离线数据指回放数据。实时传输是指在MotionStudio开启数据广播，在Unity3D中建立和MotionStudio的网络连接，就可接收到MotionStudio中的实时动捕数据。

    插件包介绍

    2.1场景介绍

    在 Assets >Mostech > Scenes 可以看到示例场景，进行简单的网络配置即可使用

    1. MotionAnimatorScene  模型是Animation Type>Humanoid类型的示例场景

    2. MotionTransformsScene 模型是Animation Type>Generic类型的示例场景

    3. MotionHandScene 驱动手指示例场景

    2.2脚本介绍

    1. 在Assets>Mostech>Scripts>MotionSourceManager.cs

      用于管理网络连接，数据驱动实例

    2. 在Assets>Mostech>Scripts>MotionConnect.cs

      具体的网络连接、数据收发、数据处理

    3. 在Assets>Mostech>Scripts>MotionData.cs

      动捕数据解析对应的实体类，以及数据转换

    4. 在Assets>Mostech>Scripts>MotionInstance.cs

      动捕数据驱动模型的基类

    5. 在Assets>Mostech>Scripts>MotionAnimatorInstance.cs

      将MotionStudio动捕数据应用于Unity的Animator组件并且是人形骨骼的模型上

    6. 在Assets>Mostech>Scripts>MotionTransformsInstance.cs

      将MotionStudio动捕数据直接应用于transform组件。适用于非人形骨架的模型。

    7. 在Assets>Mostech>Scripts>DataConvert.cs

      通用数据转换，例如:欧拉角转四元数

    8. 在Assets>Mostech>Scripts> Loom.cs

    子线程向主线程中添加事件,避免子线程直接调用unity主线程中方法报错。每个场景中必须要

    添加Loom.cs脚本。

    2.3脚本MotionSourceManager.cs公共变量说明

    1. Socket Type：服务端Socket 协议 TCP、UDP

    2. Server_ip：MotionStudio中BVH数据广播设置的Ip

    3. Server_port：MotionStudio中BVH数据广播设置的Port

    4. Client_ip：当MotionStudio协议是UDP时需要将SocketType设置为UDP，并输入本地Ip

    5. Client_port：当MotionStudio协议是UDP时,需要将SocketType设置为UDP，并输入本地port

    6. AutoConnect:是否自动连接服务端

    7. MotionInstances[]:需要被驱动的实例在这里赋值，无论是MotionAnimatorInstance.cs

    还是MotionTransformsInstance

    2.4脚本MotionAnimatorInstance.cs公共变量说明

    1. ActorID:演员的 ID 号, 默认是 0，代表 MotionStudio 中的一个角色，如果在MotionStudio

      软件中有多个角色，则第一个角色 ID 输入 0，第二个角色 ID 输入 1，依此类推。

    2. IsUseHand:是否驱动手指

    3. PositionOffset：在模型原始位置上加上这个位置偏差

    4. Animator：被驱动模型的Animator组件

    2.5脚本MotionTransformInstance.cs公共变量说明

    1. ActorID:演员的 ID 号, 默认是 0，代表 MotionStudio 中的一个角色，如果在MotionStudio

      软件中有多个角色，则第一个角色 ID 输入 0，第二个角色 ID 输入 1，依此类推。

    2. IsUseHand:是否驱动手指

    3. PositionOffset：在模型原始位置上加上这个位置偏差

    4. 51个Transform：需要手动绑定需要驱动的51个骨骼

    5. bind：一键绑定模型的各个骨骼，被绑定模型有Animator组件并且Avatar不为空

    模型要求

    1. 模型根节点旋转必须为0

    2. 模型所有骨骼自身坐标系为左手坐标系，x轴向右，y向上，z向前

    3. 左右手各个手指由三部分组成，手指近端、中端、末端组成

    4. 模型初始姿态必须为T Pose，且T pose下各个骨骼旋转最好为0

    模型T-POSE下姿态示意图

    如果模型初始姿态不是T-POSE，需要手动将模型设置为T-POSE

    模型设置为T pose有两种方式

    方式一

    模型有人形骨骼，确保每个骨骼映射正确

    方式二

    模型没有人形骨骼，手动调整每个关节

    3.模型最好有下方骨骼，满足得越多动作越精细
